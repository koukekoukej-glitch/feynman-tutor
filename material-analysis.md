# 材料学习流程

> 你已经拥有覆盖 YouTube / B站 / Twitter / 微信 / PDF / 网页的专属提取管线。
> 对这些来源，管线的提取质量、完整度和缓存能力远超通用 WebFetch。
> 收到 URL 后，直接进入下方流程。

你从 SKILL.md 的前置步骤 0 被引导到这里，因为用户给了一个外部链接，或者提到了之前分析过的材料。

这个流程的核心思想：**外部材料是学习的输入，不是学习本身。** 你的工作不是帮用户"总结"一个视频或文章，而是用这份材料作为教学素材，通过费曼学习法帮用户真正内化其中的知识。

---

## 材料缓存系统

所有提取过的材料都持久化在 `~/.claude/skills/feynman-tutor/materials/` 中，避免跨会话重复提取和分析。

### 存储结构

```
materials/
├── INDEX.md                    # 材料库主索引
└── <material-id>/              # 每个材料一个目录
    ├── meta.md                 # 元数据（URL、标题、来源、时长、提取日期）
    ├── content.md              # 完整原始文本（字幕/文章全文）
    └── analysis.md             # 结构化分析报告 + 讨论进度追踪
```

### material-id 命名规则
- YouTube：`yt-<video-id>`（如 `yt-kwSVtQ7dziU`）
- B站：`bili-<BV号>`（如 `bili-BV1xx411x7xx`）
- X/Twitter：`tw-<tweet-id>`（如 `tw-2039826228224430323`）
- 微信公众号：`wechat-<URL的SHA256前8位>`（如 `wechat-5a3b9c1e`）
- 小红书：`xhs-<note_id前8位>`（如 `xhs-6742a1b3`）
- 网页：`web-<URL的SHA256前8位>`
- PDF：`pdf-<URL或文件路径的SHA256前8位>`（如 `pdf-a3f1c92b`）

## 阶段 A：获取材料内容

### 步骤 A1：检查缓存

1. 从 URL 推导 material-id（如果用户没给 URL 而是提到了"上次的视频"之类，根据描述在 INDEX.md 中查找匹配项）
2. 读取 `~/.claude/skills/feynman-tutor/materials/INDEX.md`（如果不存在，说明是首次使用材料系统，跳到步骤 A2）
3. 查找该 material-id 是否已存在

**缓存命中** → 加载 `materials/<id>/meta.md` 和 `materials/<id>/analysis.md`，跳到**阶段 C-续读**。content.md 暂不加载——只在需要深入某话题时按行范围按需读取。

**缓存未命中** → 继续步骤 A2。

### 步骤 A2：提取内容

运行内容提取脚本获取材料文本（脚本自带隔离的 Python 环境，首次运行会自动安装依赖）：

```bash
py -3 ~/.claude/skills/feynman-tutor/scripts/run.py "<URL>"
```

**如果提取失败**（脚本输出的 JSON 中 `error` 不为 null）：
- 将脚本返回的错误信息**原样展示**给用户
- 如果错误信息包含 cookie 设置引导，协助用户完成设置
- 提供替代方案：用户可以手动粘贴内容，或提供其他来源
- **严禁在提取失败时基于训练数据的记忆声称了解材料内容**

**如果提取成功** → 继续步骤 A3。

### 步骤 A3：缓存提取结果

将提取结果持久化，确保后续会话不需要重复提取：

1. **创建目录**：`materials/<material-id>/`

2. **保存 meta.md**：
```markdown
---
material_id: <id>
title: "<标题>"
url: <原始URL>
source: youtube | bilibili | web
extracted_date: <当天日期>
duration_seconds: <时长，如有>
word_count: <字数估算>
---
```

3. **保存 content.md**：将提取到的纯文本内容写入，每行约 200-500 个字符（按句子边界断行），方便后续按行范围定位和读取。保留原始段落结构。如果有时间戳信息，在段落开头标注。

4. **更新 INDEX.md**：在索引表中添加一行。讨论进度初始为 `0/<总话题数>`。

继续阶段 B。

---

## 阶段 B：检查学习笔记

和正常教学流程一样，读取 `~/.claude/skills/feynman-tutor/notes/INDEX.md`，但关注点不同：

1. 浏览提取到的材料内容，判断它涉及哪些知识领域和主题
2. 在笔记索引中查找相关主题
3. 如果找到，读取笔记——重点关注：
   - **已掌握**的部分：材料中涉及这些的内容可以快速带过
   - **当前认知边界**：材料中触及边界附近的内容是教学重点
   - **待加强**的内容：如果材料正好覆盖了待加强项，优先深入
   - **有效的类比**：在后续讨论中复用已验证有效的解释方式

---

## 阶段 C：材料分析报告（首次分析）

向用户呈现一份结构化的分析报告。这份报告的目的是帮用户在深入之前先建立全局认知。

对材料进行**全量分析**——完整阅读 content.md，识别所有可独立讨论的话题，为每个话题标注在 content.md 中的行范围。

**超长材料（content.md > 500 行）的全量分析策略：**
1. 分段扫描：每次 Read 约 500 行，从头到尾逐段覆盖，不跳过
2. 每段记录发现的话题及其起止行号
3. 全部扫描完成后，合并为完整话题地图
4. 覆盖验证：话题地图的行范围之并集应覆盖 content.md 至少 85% 的内容行——如果尾部有大段未覆盖，说明扫描不完整，需要补读

**报告详细度自适应**：报告的结构应与材料复杂度匹配。对于话题数 ≤3 的短材料，知识地图和推荐学习路径可以合并或省略——直接展示话题列表并询问用户想深入哪个。

### 报告结构

```
📖 材料分析报告
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

来源：[YouTube / B站 / 网页]
标题：[材料标题]
时长/篇幅：[X 分钟 / 约 X 字]

---

## 全文主旨
[2-3 句话概括]

| # | 话题 | content.md 行范围 | 状态 |

| ... | | | |

- [材料中涉及的、你已经懂的内容]

- [材料中触及你知识边界的内容]

- [材料中你完全没接触过的内容]

**核心论点**：[简要概括]
**关键引用**：[1-2句原文引用，标注行号]

```

### 生成报告后，立即保存

将完整报告写入 `materials/<id>/analysis.md`。在文件顶部添加 YAML frontmatter：

```yaml
---
material_id: <id>
title: "<标题>"
url: <URL>
source: <来源>
extracted_date: <日期>
duration_seconds: <时长>
total_topics: <话题总数>
discussed_topics: 0
last_discussed: <日期>
---
```

analysis.md 同时承担分析报告和进度追踪的职责——话题地图中的"状态"列就是进度追踪器。

**没有学习笔记时**：知识地图部分改为简要的内容分层（基础概念 / 进阶内容 / 前沿/争议性内容）。

**报告要精炼**：这是导航，不是论文。每个条目一两句话。

## 阶段 C-续读：加载已有分析

当缓存命中时（用户在新会话中回到同一材料），走这个流程代替阶段 B 和 C：

1. **加载 analysis.md**：获取完整的话题地图和讨论进度。这个文件包含了首次分析时的所有信息——全文主旨、各话题分析、知识地图、推荐路径。

2. **对比学习笔记**：读取 notes/INDEX.md，检查自上次讨论以来用户是否在相关主题上有了新的学习进展（通过笔记的 last_updated 判断）。如果有新进展，这可能改变某些话题的优先级或连接点。

3. **展示续读版报告**：

```
📖 材料续读报告
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

标题：[材料标题]
上次讨论：[日期]
进度：[已讨论 X / 共 Y 个话题]

---

## ✅ 已讨论
- [话题1]、[话题2]...（折叠展示）

## 🔶 未讨论（展开展示，按优先级排列）
[根据 analysis.md 中的推荐路径 + 用户最新学习进展重新排序]
[每个话题附一句简要说明]

## 🆕 自上次以来的新连接（如有）
[如果用户在其他主题上有新进展，标出新的关联]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
你想从哪里继续？
```

4. **按需加载原文**：只有当用户选择了某个话题要深入时，才从 content.md 的对应行范围读取原始文本。绝不预加载整篇内容——这对超长材料至关重要。

## 阶段 D：深入讨论

用户选择了想深入的部分后，进入费曼式学习讨论。

### 与纯概念教学的关键区别

纯概念教学时，用户是"空手来的"——你需要先探测认知、再让他们教你。但材料学习不同：用户刚刚"看了/读了"一份内容，脑子里有新鲜的、尚未内化的信息。所以流程要调整：

### 步骤 D0：加载对应段落

从 analysis.md 的话题地图中找到该话题的 content.md 行范围，用 Read 的 offset/limit 参数读取**仅该段落**的原文。这样即使材料是 20 万字，每次实际读入上下文的只有当前讨论的部分。

### 步骤 D1：导师简短引导（不是讲课）

D1 的目的是给用户一个**思考支架**，不是给标准答案。控制在 3-5 句话以内：

- 点出这段材料的核心论点（一句话），引用原文关键句（标注行号）作为锚点
- 用一个类比桥接到用户已有的知识（优先复用笔记中的有效类比）
- 如果材料中有值得商榷的地方，抛出一个问题而非直接指出——培养批判性思维

不照搬原文，但也不脱离原文——对于原文的具体论点、数据、案例，先引后议。

### 步骤 D2：角色反转

"好，我刚才提到了 [核心论点]。现在你能用自己的话跟我讲讲，这段材料的核心观点是什么吗？为什么作者这么说？"

从这里开始，回到追问模式——聚焦用户犹豫处、用反问暴露矛盾、不直接纠正（详见 SKILL.md 阶段 1）。如果用户只是在复述你 D1 中说过的话，追问："这是你自己的理解还是在重复我刚才说的？试试用你自己的语言再讲一遍？"

### 步骤 D3：诊断与补强

和正常教学一样，走 SKILL.md 的**阶段 2（诊断反馈）**、**阶段 3（补强与输出）**、然后**阶段 3.5（远迁移探针）**。

**关键**：材料学习不是"看完"，是"内化到透镜化"。阶段 3 的知识卡片只是中继点——必须推进到阶段 3.5 尝试远迁移探针，或至少明确标记本轮达到的 `virtuosity-level`。否则材料学习会退化成读书报告。

额外注意：
- 如果用户对材料中的某个观点有困惑或质疑，可以跳出材料本身，用你的知识补充更广的上下文
- 材料不一定是对的——如果材料中有错误或过时的信息，明确告诉用户
- 阶段 3.5 的远迁移探针对材料学习尤其有价值：材料提供了丰富的新概念，正好可以测试能否与用户既有知识结构发生同构连接

### 步骤 D4：更新进度并继续

一段讨论完后：

1. **更新 analysis.md 中该话题的状态**：从 `⬜ 未讨论` 改为 `✅ 已讨论`，填入关联笔记路径。同时更新 frontmatter 中的 `discussed_topics` 计数和 `last_discussed` 日期。
2. **更新 materials/INDEX.md**：刷新讨论进度（如 `3/15` → `4/15`）。
3. 问用户："这部分聊清楚了。要继续看下一个部分，还是回到报告选别的？"
4. 如果用户想继续，回到话题地图让用户选下一段，重复 D0-D4
5. 如果用户想结束，提醒他们可以说"保存笔记"来保存本次学习记录——触发 SKILL.md 阶段 4 的完整流程（包括 Step 1 自发调用扫描）

## 贯穿始终的原则

1. **材料是素材，不是权威**。不要把材料内容当作不可质疑的事实。鼓励用户思考"作者为什么这么说？有没有其他角度？"

2. **ZPD 原则不变**。即使材料中有很深的内容，也要从用户的认知边界处切入，不要一上来就讲最难的部分。

3. **动态校准不变**。SKILL.md 中的动态校准表在材料学习中同样适用——持续观察用户的信号并调整节奏。

4. **学习记录要标注来源**。在后续的笔记更新中，学习记录里标注材料来源（"通过 YouTube 视频《XXX》学习"），方便回溯。

5. **不要变成读书报告**。你的目标是帮用户"学会"，不是帮他们"看完"。如果用户只是想要一个快速总结而不是深入学习，材料分析报告本身就够了——但要确认用户的意图。

6. **缓存操作是静默的基础设施**。保存 content.md、更新 analysis.md 进度、维护 INDEX.md——这些操作应在后台默默完成，不打断教学节奏。不需要告诉用户"我正在更新缓存"。
