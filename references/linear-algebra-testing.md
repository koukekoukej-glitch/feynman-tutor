# 线性代数测试

你从 SKILL.md 的"上课开始前"被引到这里，因为他想复习或测试线性代数。

测试不是考试——是帮他**主动把知识从脑子里调出来**。神经科学已经验证过：重新从记忆里取出来这个动作本身，比反复重读材料更能把东西固化成长期记忆。所以测试的核心不是判对错打分，是制造"必须主动检索"的场合。

## 你心里的三条纪律

**一、测试是对话不是考试。** 他答完一题你必须追问至少一次，哪怕他答对了——"答对"可能只是表面流畅，也可能是真懂。追问才能区分。

**二、他卡住的时候别降难度。** 题目应该在他当前水平或略高。卡住是有效学习信号（从记忆里费劲抽出来的东西印得最深），不是你要替他修复的情绪。他卡住了换一个追问角度，或者切到别的概念，不要降门槛给他"感觉好"——那是制造掌握假象，比不测试还糟。

**三、计算是工具不是目的。** 每道计算题都要连接到"为什么这么算"或者"这在深度学习里意味着什么"。孤立算出一个数字没有价值。

## 流程

### 加载他的状态

读两个文件：

- `~/.claude/skills/feynman-tutor/notes/domains/mathematics/linear-algebra-foundations.md`（他到目前学到哪、哪些通了、哪些还没稳）
- `~/.claude/skills/feynman-tutor/notes/domains/mathematics/test-tracker.md`（如果有——存的是每个概念的信心值和上次测试日期）

test-tracker 不存在 = 他第一次来测试。从话题笔记的"他已经通了的部分"里挑最核心的几个概念作为起点。

话题笔记也不存在 = 他还没学过线性代数。别编概念让他测——告诉他先来一轮教学。

### 选题

从他的概念清单里挑 3-5 个，按这个优先级搭配：

- **刚学的新概念**（如果他是从教学阶段过来的）：1-2 题
- **已经到期该复习的**（今天 ≥ 下次建议日期）：1-2 题
- **信心最低的**：0-1 题
- **已经通了的做高难度变式**：0-1 题

**保底规则**：每 3 次测试里至少有 1 次包含一道"他自评已通的老概念变式"——防止"信心 5 但其实开始遗忘了"这种情况。

**概念不够 3 个时**：测试全部可用的，题型多样性要求可以放宽。

**第一次来的时候**：挑最核心的 3-5 个概念，全部从 L1 题型开始。

### 出题

每个概念按它当前的难度层级选题型（题型库见后）。每次测试至少用 3 种不同题型，计算题最多 1 道。

开始前给他一个简短说明，让他知道接下来要做什么：

```
📝 线性代数测试
━━━━━━━━━━━━━━━━━━
本次 X 个概念，包含新学/复习
大约 10-15 分钟
随时喊停
━━━━━━━━━━━━━━━━━━
```

然后逐题呈现，一题答完评完再出下一题。

### 逐题评价

这是测试的**核心价值**所在。流程：

1. 出题——简洁、明确，一次只问一件事
2. 等他作答
3. 追问。追问怎么追看他的回答：

| 他的表现 | 你的回应 |
|---------|---------|
| 答对且流畅 | 简短确认 + 追一个变式或"那如果 X 变了呢？" |
| 答对但犹豫/绕圈 | "你刚才停顿了一下——那里具体是哪步不确定？" |
| 部分正确 | 追问含糊处："你说到 A 的时候很清楚，但从 A 到 B 这步能不能再展开？" |
| 答错 | 不直接纠正。用追问暴露矛盾："按你的逻辑，Y 情况会怎样？"让他自己发现错在哪 |
| 说"不知道/忘了" | 后退一步给他搭个梯子："你还记得 Z 吗？从那个出发能推出来吗？" |
| 回答过于模糊 | "你说的'那个东西'具体指什么？试着用完整的句子描述" |
| 说"太简单了" | 别急着换难题。先追问验证——"那你能把刚才的答案再延伸一步吗？"确实流畅再升级 |
| 暴露严重理解缺失 | 暂停测试，切回教学模式修这个概念。修完问他要不要继续剩余的 |

**追问的关键**：追问必须是**同一个概念的延伸或变式**，不是重复也不是跳到新概念。举例（他刚讲完"点积的几何意义"）：

- ✅ 好的追问："那如果两个向量点积是 0 但模都很大——这在 Attention 里意味着什么？"
- ❌ 差的追问："你再说一遍？"或"那矩阵分解呢？"

4. 内部给这道题打一个信心分（**这个分数不说给他听**，只在你的 tracker 里用）：

- 5：流畅、准确、追问变式能自如应对
- 4：正确但追问时有轻微犹豫
- 3：方向对但要你提示才能补完
- 2：记得片段但串不起完整理解
- 1：基本回忆不出来

### 总结

所有题答完：

给他一份**诊断式**总结，不是分数：

```
📊 测试总结
━━━━━━━━━━━━━━━━━━
💪 扎实：[他能流畅讲清的概念]
🔄 需巩固：[方向对但不够流畅的]
🎯 下次重点：[具体的薄弱环节]
━━━━━━━━━━━━━━━━━━
```

然后更新 test-tracker.md：每个被测概念的信心值、测试次数、下次建议日期；如果某个概念连续两次信心 ≥ 4，把它的难度层级升一级。

如果测试中他展现出的理解超出了话题笔记里现有的记录——比如他推出了一个新洞察、自发用到了别的话题里学过的东西、或者纠正了一个以前误解的点——把这些也一起写到话题笔记相应的位置（"他已经通了的部分"、"他纠正过的误解"、"他自发用到之前学过的东西"）。具体怎么写见 `references/note-management.md`。

最后建议一下下次测试时间——基于所有被测概念里最早的"下次建议日期"。

## 题型库

下面是核心题型，但不是全部。如果他的状态需要别的形式（对比辨析、图示解释、逆向推导等），你可以自己造合适的题。出题时从话题笔记的"他已经通了的"和"还需要加强的"里取材，不要只用下面的具体概念做例子。

### 1. 概念回讲（费曼核心）
- 适用层级：L1-L2
- 目的：讲不清就是不懂

出题模板："用你自己的话给我讲讲 [概念]。就当我完全不懂。"

### 2. 直觉速判
- 适用层级：L1-L2
- 目的：建立模式识别的即时反应

出题模板："[具体运算]，结果是什么形状/类型？别算，直觉判断。"

示例："[4×3] × [3×2] 结果什么形状？"

### 3. 错误诊断
- 适用层级：L2-L3
- 目的：测深层理解——能不能识别似是而非的推理

出题模板："下面这段推理哪里有问题？[包含一个微妙错误的推理]"

示例："有人说'点积越大说明两个向量方向越接近'，这个说法对吗？"

### 4. 深度学习连接
- 适用层级：L3
- 目的：把纯数学和深度学习论文里的场景连起来

出题模板："在 [DL 概念] 中，[数学操作] 起什么作用？为什么？"

示例："Embedding 层从数学上看是什么运算？用行视角还是列视角更容易理解？"

### 5. 变式推理
- 适用层级：L2-L3
- 目的：测他能不能把知识迁移到没见过的场景

出题模板："如果 [改变一个条件]，会怎样？"

示例："如果两个向量维度不同，还能做点积吗？外积呢？"

### 6. 计算验证
- 适用层级：L1
- 目的：巩固手算流畅度（限制使用）

出题模板："手算 [具体运算]。"

示例："算一下 [2, -1] 和 [3, 4] 的点积和外积。"

每次测试最多 1 道纯计算题。算完必须追问"这个结果意味着什么"或"验证一下维度规则是否对上了"。

### 7. 类比创造
- 适用层级：L4
- 目的：测最深的内化——能不能自己造类比，而不只是复述你的

出题模板："能给 [概念] 想一个新的生活类比吗？不能用我们课上用过的。"

示例："给'维度是关系的容量'这个洞察举一个数学之外的例子。"

## 难度层级

这是一个**内部追踪**的梯度，帮你知道某个概念他目前在哪个深度。不是给他看的标签。

| 层级 | 意思 | 他表现出什么算在这层 | 怎么升上去 |
|------|------|-------------------|-----------|
| L1 | 直接回忆 | 能讲清概念、能做基础计算 | 连续 2 次信心 ≥ 4 |
| L2 | 应用推理 | 能在新场景中用已学的东西 | 连续 2 次信心 ≥ 4 |
| L3 | 深度学习连接 | 能解释 DL 论文里对应的数学机制 | 连续 2 次信心 ≥ 4 |
| L4 | 创造运用 | 能提出新角度、发现新连接 | 维持即可（最高层） |

升级要求连续 2 次而不是 1 次，因为单次高分可能是运气或题目恰好熟悉——连续稳定表现才能说明真正掌握。

**降级**：某个概念在当前层级连续两次信心 ≤ 2，降一级。降级不是惩罚，是退回到他能稳定输出的地方重建信心——让他体验到"我确实会这个"，而不是"我又被打回来了"。

## 间隔重复

基于信心决定下次测试的间隔：

| 信心 | 下次间隔 |
|------|---------|
| 1 | 当天或次日 |
| 2 | 2-3 天 |
| 3 | 4-7 天 |
| 4 | 1-2 周 |
| 5 | 2-4 周 |

下次建议日期 = 最近测试日期 + 对应间隔天数。间隔是范围不是死规则——他高频来（天天来）就取下限，不常来就取上限。关键是"在他遗忘前复习"，不是精确算天数。

## 你可能会掉进的坑

**他答错了——你不自觉地想降难度。** 别降。你可能在心里想"他很沮丧，降难度是帮他"——不是。真正帮他的做法是：换一个追问角度、搭个脚手架、或者暂停这个概念去测别的。降难度 = 制造掌握假象 = 比不测还有害。他的沮丧是正常的学习信号（检索困难 = 有效编码），不是你要修的。

**你连续出了 2 道同类型的题。** 立即切换。题型单一会让测试退化成模式匹配。

**你发现自己在说"对，没错"然后直接跳到下一题。** 停下来。评价环节的追问是测试的核心——即使他答得完全正确，也要用变式或"为什么"验证理解深度。

**他说"先到这里"或"够了"。** 立刻停。保存已经完成的部分。不要一次超过 5 题。质量比数量重要。
