# 推理工程测试

你从 SKILL.md 的"上课开始前"被引到这里，因为他想复习或测试 LLM 推理工程与 Kimi 技术体系的内容。

测试不是考试——是帮他**主动把知识从脑子里调出来**，并且练习把基础设施知识和产品决策连起来。这个话题和线性代数测试不同：没有纯计算题，核心在于他能不能从底层机制出发推导出工程约束、再从约束推导出产品决策。

## 你心里的三条纪律

**一、测试是对话不是考试。** 他答完一题你必须追问至少一次，哪怕他答对了——"答对"可能只是表面流畅，也可能是真懂。追问才能区分。

**二、他卡住的时候别降难度。** 卡住是有效学习信号。换一个追问角度、切到别的概念，不要降门槛给他"感觉好"。

**三、每个概念最终都要连到产品。** 孤立背诵"decode 瓶颈在带宽"没有价值。追问"这对产品设计意味着什么"或"如果你是 PM 你会怎么利用这个约束"。

## 题型库

### 1. 概念回讲（费曼核心）
- 适用层级：L1-L2
- 目的：讲不清就是不懂

出题模板："用你自己的话给我讲讲 [概念]。就当我完全不懂 GPU 是什么。"

示例："给我讲讲为什么 decode 阶段的瓶颈在显存带宽而不在算力。"

### 2. 机制追踪
- 适用层级：L1-L3
- 目的：验证他能不能从头到尾串起一个完整过程

出题模板："描述一下 [某个过程] 的完整步骤。"

示例：
- L1："一个用户发了一条消息，从收到消息到第一个字输出，GPU 上发生了什么？"
- L2："在 Mooncake 架构下，一个 128K token 的长文档请求是怎么处理的？"
- L3："在 MoE 模型里，一个 token 经过一个 MoE 层的完整过程是什么？包括通信。"

### 3. 瓶颈诊断
- 适用层级：L2-L3
- 目的：测他能不能识别特定场景下的关键瓶颈

出题模板："在 [具体场景] 下，系统的瓶颈在哪？"

示例：
- "上下文 100 万 token、decode 阶段，主要瓶颈是什么？FlashAttention 能解决吗？"
- "MoE 模型的激活参数只有 3%，理论上计算量很小，但实际推理速度不一定比同等激活参数的稠密模型快——为什么？"

### 4. 产品决策推理
- 适用层级：L3-L4
- 目的：测他能不能从技术约束推导出产品决策

出题模板："如果你是 PM，在 [技术约束] 下你会怎么设计 [产品功能]？"

示例：
- "KV Cache 前缀复用能让共享前缀的请求几乎零成本。你会怎么利用这个特性设计产品功能？"
- "GPU 资源不够服务所有用户时，你会怎么设计用户优先级策略？从技术约束出发推理。"
- "Mooncake 的过载调度在技术上已经能预测性拒绝请求了，但用户端的体验还需要产品设计。你会怎么处理'被拒绝'的用户？"

### 5. 对比辨析
- 适用层级：L2-L3
- 目的：测他能不能区分容易混淆的概念

出题模板："[A] 和 [B] 各自解决什么问题？能互相替代吗？"

示例：
- "FlashAttention 和线性注意力各自解决推理的什么问题？"
- "张量并行和流水线并行的区别是什么？什么时候用哪个？"
- "MoE 的 all-to-all 通信和 all-reduce 通信有什么本质区别？"

### 6. 错误诊断
- 适用层级：L2-L3
- 目的：测深层理解——能不能识别似是而非的推理

出题模板："下面这段描述哪里有问题？"

示例：
- "有人说'FlashAttention 通过近似计算让 attention 从 O(N²) 降到了 O(N)'，这个说法对吗？"
- "有人说'MoE 模型因为每个 token 只用 3% 的参数，所以只需要 3% 的显存'，哪里有问题？"
- "有人说'GPU 给 20 个用户做推理时，每个用户的等待时间是 1 个用户的 20 倍'，对不对？"

### 7. 量级估算
- 适用层级：L2-L3
- 目的：建立对实际数量级的直觉

出题模板："粗略估算 [某个量]。"

示例：
- "一个 70B 参数的模型用 FP16 存储需要多少显存？需要几块 A100？"
- "上下文 128K token 的一个用户，KV Cache 大约多大？（给出每层的 K 和 V 维度）"

## 难度层级

| 层级 | 意思 | 他表现出什么算在这层 | 怎么升上去 |
|------|------|-------------------|-----------|
| L1 | 直接回忆 | 能讲清概念、能串起基础机制链 | 连续 2 次信心 ≥ 4 |
| L2 | 应用推理 | 能在新场景中识别瓶颈、做对比辨析 | 连续 2 次信心 ≥ 4 |
| L3 | 产品连接 | 能从技术约束推导出产品决策 | 连续 2 次信心 ≥ 4 |
| L4 | 综合创造 | 能提出新角度、发现跨论文的非显然连接 | 维持即可 |

升级要求连续 2 次而不是 1 次。降级规则同线性代数测试。

## 概念分类

追踪器里的概念按以下分类组织，随着学习新论文会不断扩展：

**GPU 推理基础**
- GPU 批处理与权重共用
- Decode 阶段带宽瓶颈
- GPU 显存内容与存储层次

**模型部署**
- 张量并行与流水线并行
- MoE 专家结构与路由
- MoE all-to-all 通信
- 量化（FP16/INT8/INT4）

**注意力优化**
- FlashAttention 与在线 Softmax
- FlashAttention vs 线性注意力 vs 稀疏注意力的定位
- KV Cache 生成与前向计算的关系
- Prefill 分块流水线 vs Decode 串行

**推理服务架构（Mooncake）**
- Prefill/Decode 分离
- KV Cache 分布式缓存与前缀复用
- 过载调度三层递进

**产品映射**
- KV Cache 复用的产品经济学
- 长上下文用户的竞争壁垒
- 用户优先级与 SLO 设计
