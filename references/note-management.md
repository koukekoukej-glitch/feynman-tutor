# 怎么做笔记

你从 SKILL.md 的"讨论结束后"被引到这里。一场对话走到自然结束，你要把今天的东西记下来。这份文档讲每种文件要写什么、怎么更新。

笔记是给下次的你看的——写稀了下次接不上，写多了下次读得慢。

---

## 笔记放在哪

所有笔记在 `~/.claude/skills/feynman-tutor/notes/` 下：

```
notes/
├── INDEX.md                   # 导航索引
├── learner-core.md            # 跨主题看得清的关于他的东西
├── learner-history.md         # 按时间线的学习轨迹
├── cross-domain/              # 跨领域模式（每个一个文件）
│   ├── 不可逆信息压缩.md
│   └── ...
└── domains/{领域名}/          # 按领域分组
    ├── domain.md              # 这个领域的整体 checklist
    └── {话题名}.md            # 具体话题笔记
```

---

## 走完一场对话之后

### 1. 整理今天的东西

先把这场对话过一遍，心里整理出：

- **主题**：今天聊的是什么
- **领域**：属于哪个知识领域
- **他今天通了的部分**：他自己讲清楚了的（不只是你讲了的）
- **他纠正了的误解**：原来怎么想 → 实际怎么回事
- **当前他停在哪里**：下次应该从哪里接着
- **还需要加强的**：今天提到了但没深入的东西
- **今天对他有效的类比或解释**：下次可以复用
- **他今天自发用到之前学过的东西**（有就记，没有就跳过）：他在讨论 A 话题时用了 B 话题里的概念来解释

### 2. 看一下之前的笔记

读 `notes/INDEX.md`（如果存在）：
- 查今天的主题是不是之前已有笔记
- 有 → 更新那一份（不要新建）
- 没有 → 确定属于哪个领域，准备在 `domains/{领域名}/` 下新建

### 3. 展示给他看

用下面的格式给他看一份总结：

```
📓 今天的学习总结
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
主题：[主题名]
领域：[领域]
操作：[新建 / 更新]

✅ 他今天通了的部分：
- ...

🔄 他纠正过的误解：
- 原以为 [错]
- 实际上 [对]

📍 当前边界：
[下次从哪里接]

📋 还要加强的：
- ...

💡 今天对他有效的解释/类比：
- ...

🌱 他今天自发用到之前学过的东西（有就有，没有就不写这节）：
- 在讨论 [当前话题] 时用了 [之前学过的 X]
  → 会在 X 的笔记里加一条记录
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

问他一句："这份总结准确吗？要改或补什么吗？确认后我就保存。"

**等他明确说"可以"/"对"/"保存"再写。他没确认就别写。** 他可能会要你改——先按他说的改，再重新问一次。

### 4. 确认之后写入

按下面的格式写文件：

**话题笔记格式：**

```markdown
---
topic: [主题名]
domain: [领域目录名]
last_updated: [YYYY-MM-DD]
status: [in-progress / completed]
cross-domain-patterns:
  - [相关跨领域模式文件名，如果有]
related-topics:
  - [相关话题文件名，如果有]
---

## 他在这个主题上现在站在哪里
[一段话，像你跟同事交接学生情况——他哪些地方已经通了、哪些地方还在边缘、下次从哪里接。]

## 他已经通了的部分
- [概念 1]：用他自己的话记录他的理解
- [概念 2]：...

## 他纠正过的误解
- 原以为... 实际上...

## 当前边界
下次对话应该从哪里继续。

## 还需要加强的
- [概念]：为什么需要加强

## 对他有效的类比和解释
- [类比]：适用场景

## 学习记录
- [YYYY-MM-DD]：这次的简要摘要

## 他自发用到之前学过的东西
（仅当某次对话中真的出现了这种情况时记；没有就不要这节）
- [YYYY-MM-DD]：在讨论 [当前话题] 时用 [之前学过的 X] 解释/预测了什么，用得对不对
```

**更新已有笔记的规则：**

- **他已经通了的部分**：追加新通的，保留之前的
- **他纠正过的误解**：追加新的
- **当前边界**：更新为最新
- **还需要加强的**：某项这次学会了就挪到"他已经通了的部分"；追加新的待加强项
- **学习记录**：追加本次
- **他在这个主题上现在站在哪里**：根据整体进展更新
- **他自发用到之前学过的东西**：追加

### 5. 更新几个一定要动的文件

**每次必做：**

| 文件 | 动什么 |
|------|--------|
| 今天的话题笔记 | 新建或更新 |
| `learner-core.md` 的 frontmatter | `sessions_count` +1；`last_updated` 改今天 |
| `learner-history.md` | 追加本次的一行记录 + 更新 frontmatter |
| 今天话题所在的 `domains/{领域}/domain.md` | 在 checklist 里标 ✅，更新 `topic-count` |

**按情况做：**

| 情况 | 要做的 |
|------|--------|
| 新建话题 | 在 INDEX.md 话题索引表追加一行；对应领域的话题数 +1 |
| 新建领域 | 创建 `domains/{新领域}/` 目录 + `domain.md`；在 INDEX.md 领域目录表追加一行 |
| 他今天自发用到了别的话题里的概念 | 去那个**被用到的话题**的笔记里，在"他自发用到之前学过的东西"那节追加一条；同时更新那个话题的 `last_updated` |
| 今天学的东西和其他领域有结构上的共通点 | 看 `cross-domain/` 里是不是已经有这个模式。有就追加新实例；没有但确实在 ≥2 个领域都出现过，才考虑新建一个 `cross-domain/*.md` 文件 + 在 INDEX.md 跨领域模式索引表追加一行 |
| 今天看到了关于他思考方式的新模式，而且之前其他话题里也见过 | 加到 `learner-core.md` 的"他是怎么想事情的"或"你注意到了但还没看清的"——看证据强度（见下一节） |

**learner-history.md 的格式：**

在文件末尾追加：`- **YYYY-MM-DD**（第 N 轮，主题名）：一段话描述本次学习的路径和标志性特征`。同时更新 frontmatter。

**INDEX.md 的结构：** 四表——元文件、领域目录、话题索引、跨领域模式索引。按现有格式追加即可。

**domain.md 格式（新建领域时）：**

```markdown
---
type: domain
domain: {领域名}
topic-count: 1
---

### {领域人类可读名称}

**[子领域/方向名]**
- ✅ [本次学习的话题/概念]
```

## 关于 learner-core.md 要怎么动

`learner-core.md` 是三层结构（见那个文件本身）。每次写这个文件时分清楚：

### 什么放"对话协议"
**只有他明确向你提出的要求才能进这里。** 你观察到的不算。比如他说过"不要用代码术语"、"不要用类比"——这些是他明确告诉你的，进这里。

### 什么放"他是怎么想事情的"
跨主题看得清楚的、稳定的关于他的模式。每一条都要能说清楚：是哪几条证据让你看到了这件事？每条证据要带上他的原话或具体场景。

升级到这里之前要问自己：
- 这个模式**只在单次对话**出现过吗？——那不算看清，放到下一栏
- 我说得清**是哪几条不同的证据合在一起**指向同一件事吗？——说不清就放下一栏
- 这里面我有没有**超出证据本身的推断**？——有就删掉推断，只留证据能直接支撑的部分

### 什么放"你注意到了但还没看清的"
这里**不做推论、不起名字、不归类**。只记"在什么场景下他做了什么"。例如："2026-04-21 讨论 X 时，他做了 Y 事情——值得留意"。

后来的对话积累下来，能让你真正看清"这是他的某某模式"的时候，再从这一栏挪到上面那栏。转不过去就留着，没关系。

**永远不要从一次观察直接升级成画像。**

## 关于 cross-domain（跨领域模式）

这一类笔记只在真有跨领域共通点时才做。不要为了填而填。

几个判断：

- **已有模式扩展**：今天学的东西是不是给某个已有的 `cross-domain/*.md` 提供了新实例或新维度？——追加到那份文件
- **新模式发现**：今天学到的某个机制/原则，是不是在 ≥2 个已学领域都出现过？——那才考虑新建一个模式文件
- **冲突/张力**：今天学的"最佳实践"是不是在另一个领域里是反模式？——这也值得记，在相关的两份笔记里互相指出
- **有没有更好**：今天有没有哪种思维方式从另一个领域迁移过来特别好用？

**重要：每次最多 1-2 个高质量连接。** 质量远比数量重要。没有值得记的就别硬找。

连接必须附带"为什么"——不是"A 和 B 相关"，而是"A 中的 X 机制和 B 中的 Y 机制在结构上是同一个东西，因为……"

## 给他看的原则

- **用他自己的话记录。** 笔记的读者是下次那个你。你要忠实反映他实际怎么想，比用教科书术语更有价值。
- **认知水平基于他对话里的实际表现判断。** 他能流畅讲出来的才算"通了"；你讲过但他没有回应验证的只能算"待加强"。
- **笔记要精炼。** 每个条目一两句话概括，不要复制对话原文。笔记是索引，不是抄本。
- **一次对话可能涉及多个主题。** 如果跨了多个领域，为每个主题分别做笔记，逐个展示总结。
