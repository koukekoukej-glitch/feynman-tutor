# 费曼学习笔记管理

你从 SKILL.md 的收尾阶段被引导到这里，因为一轮学习对话即将结束，需要将学习成果持久化。

你的职责是从对话中提取有价值的学习信息，维护一套持久化的学习笔记。这套笔记是导师了解学生的核心工具——它让导师在下一次对话中能精准地知道学生的认知水平、学过什么、哪里有盲点，从而避免重复教已经会的东西，也不会跳过还没掌握的基础。

## 笔记存储

所有笔记存储在 `~/.claude/skills/feynman-tutor/notes/` 目录下：
- `INDEX.md`：笔记索引，列出所有主题及其状态
- 每个主题一个独立的 md 文件（如 `attention-mechanism.md`）
- `LEARNER.md`：学习者模型——跨主题的认知风格、习惯性盲区、有效策略
- `GRAPH.md`：认知地图——跨主题的知识连接、领域框架、前沿探索区

## 触发后的流程

### 第一步：收集信息

回顾当前对话，提取以下信息：

- **主题**：这轮对话学习了什么
- **领域**：属于哪个知识领域
- **起始水平**：对话开始时用户的认知水平
- **已掌握**：用户在本次对话中展示出扎实理解的概念（重点是用户自己讲出来的、讲对的部分，而不只是导师教过的）
- **已纠正的误区**：用户最初的错误理解，以及纠正后的正确理解
- **当前认知边界**：用户目前的理解停在哪里，下次应从哪里继续
- **待加强**：提到但未深入的概念，或用户表示还需进一步学习的内容
- **有效的类比**：对话中特别有效的类比或解释方式（这些是经过验证的——下次可以复用）

### 第二步：检查已有笔记

读取 `~/.claude/skills/feynman-tutor/notes/INDEX.md`（如果存在）：
- 如果已有相关主题的笔记，读取该笔记，准备**更新**而不是新建
- 如果是全新主题，准备**新建**笔记

### 第三步：展示总结

向用户展示一份结构化的学习总结：

```
📓 学习笔记总结
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
主题：[主题名]
领域：[领域]
认知水平：[入门 / 进阶 / 掌握]
操作：[新建 / 更新]

✅ 已掌握：
- ...

🔄 已纠正的误区：
- ❌ [原来的错误理解]
- ✅ [纠正后的理解]

📍 当前认知边界：
...

📋 待加强：
- ...

💡 有效的类比/解释：
- ...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

然后问用户："这个总结准确吗？有需要补充或修改的地方吗？确认后我就保存。"

**等待用户确认。不要在用户确认之前写入任何文件。**

### 第四步：用户确认后保存

用户确认后（可能会要求修改，先按要求修改再保存），写入笔记文件。

**笔记文件格式：**

```markdown
---
topic: [主题名]
domain: [领域]
level: [beginner/intermediate/advanced]
last_updated: [YYYY-MM-DD]
status: [in-progress/completed]
---

## 认知画像
[一段话描述用户在这个主题上的整体认知状态，用自然语言写，像是一个老师在跟同事交接学生情况]

## 已掌握
- [概念1]：[用用户自己的语言描述的理解]
- [概念2]：...

## 已纠正的误区
- [误区1]：原以为... → 实际上...

## 当前认知边界
[描述用户目前的理解停在哪里，下次对话应该从这里继续]

## 待加强
- [概念/技能]：[为什么需要加强]

## 有效的类比
- [类比1]：[适用场景]

## 学习记录
- [YYYY-MM-DD]：[本次学习的简要摘要]
```

**更新已有笔记时的规则：**
- 「已掌握」：追加新掌握的内容，保留之前的
- 「已纠正的误区」：追加新发现的误区
- 「当前认知边界」：更新为最新的边界位置
- 「待加强」：如果某项已经在本次学会了，移到「已掌握」；追加新的待加强项
- 「学习记录」：追加本次记录
- 「认知画像」和 `level`：根据整体进展更新

**更新 INDEX.md：**

```markdown
# 费曼学习笔记索引

| 主题 | 领域 | 水平 | 状态 | 最后更新 | 文件 |
|------|------|------|------|----------|------|
| 注意力机制 | AI/深度学习 | intermediate | in-progress | 2026-03-22 | attention-mechanism.md |
```

### 第五步：确认保存

保存完成后，简短告知用户笔记已保存，然后**立即进入第六步**。

### 第六步：认知地图更新

保存主题笔记后，更新认知地图。这一步的价值在于：让跨主题的认知连接和元认知模式显性化，使下次学习任何新东西时，导师都能从用户的认知全景出发。

**流程：**

1. 读取 `LEARNER.md` 和 `GRAPH.md`。如果这两个文件不存在：
   - 检查 `notes/INDEX.md` 中已有多少个主题笔记
   - **≥2 个主题笔记** → 触发初始化：从已有笔记中提取跨主题认知模式创建 LEARNER.md，建立已有主题之间的连接创建 GRAPH.md，然后正常执行本步骤后续流程
   - **< 2 个主题笔记** → 跳过本步骤（跨主题分析需要至少 2 个主题作为基础）

2. **前置门控：快速判断是否值得扫描。** 本次学习内容和已有知识版图有明显交叉吗？（判断依据：主题领域是否相邻、对话中是否自然涌现过跨主题联想、GRAPH.md 中有没有涉及相近领域的已有连接）
   - **有明显交叉** → 继续下方详细扫描
   - **没有明显交叉** → 跳过步骤 3-4 的扫描，直接告知用户"本次学习内容相对独立，暂未发现有价值的跨主题连接"，然后跳到步骤 5

3. **扫描跨主题连接**——将本次学习的新内容与 GRAPH.md 中已有的连接和其他主题做对比，寻找：
   - **结构类比**：本次学到的某个机制/关系模式，和已学过的其他主题中的某个机制/关系模式同构吗？（这是价值最高的连接类型）
   - **共同模式**：本次学到的某个原则/规律，在已学的其他领域中也出现过吗？
   - **张力/矛盾**：本次学到的某个"最佳实践"，在另一个已学领域中恰好是反模式吗？
   - **认知迁移**：本次对话中，有没有从旧领域迁移过来的思维方式或类比特别有效？
   - **前置条件**：学完这个之后，哪些新领域变得可学了？

4. **扫描学习者模型变化**——对比本次对话中观察到的认知模式与 LEARNER.md 的记录：
   - 有没有新的跨主题认知风格被观察到（需要在 ≥2 个主题中出现才值得记录）？
   - 有没有新的习惯性盲区暴露出来（与已记录的盲区是否同类）？
   - 有没有新的有效策略被验证？
   - 成长轨迹有没有标志性事件值得记录？

5. **更新领域框架**——在 GRAPH.md 的"领域框架"中，将本次学习覆盖的新概念标记为 ✅

6. **向用户展示发现**——只展示有实质意义的发现（不要为了填充而捏造连接）：

```
🔗 认知地图更新
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📌 新发现的跨主题连接：
- [连接类型]：[简述同构关系]
  涉及：[主题A] ↔ [主题B]

🧠 学习者模型更新：
- [新观察到的模式/盲区/策略]

📊 领域框架：
- [领域名] 已覆盖区域更新

确认后我就更新认知地图。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**等待用户确认。** 用户可能会修正、补充或拒绝某些发现。

7. **确认后写入** GRAPH.md 和/或 LEARNER.md。

**关键原则：**
- **每次只提 1-2 个高质量连接**，质量远比数量重要。不要为了展示"发现了很多"而降低门槛。如果本次确实没有值得记录的跨主题连接，坦诚告知并跳过。
- **连接必须附带"为什么"**——不只是"A 和 B 相关"，而是"A 中的 X 机制和 B 中的 Y 机制具有同构性，因为……"
- **学习者模型只记录跨主题模式**——单一主题内的认知状态留在主题笔记的"认知画像"中。

最后告知用户："笔记和认知地图都已更新。下次学新东西时，我会自动加载你的认知全景。"

## 重要原则

- **用用户自己的语言记录。** 笔记的读者是下一次对话中的费曼导师，目的是让导师快速了解这个学生的实际认知状态。忠实反映用户的理解方式，比用教科书术语更有价值。
- **认知水平基于对话中的实际表现判断。** 用户能流畅教出来的才算"已掌握"，导师讲过但用户没有回应验证的只能算"待加强"。
- **一次对话可能涉及多个主题。** 如果对话跨了多个领域，为每个主题分别创建/更新笔记，并逐个展示总结。
- **笔记要精炼。** 每个条目用一两句话概括，不要复制对话原文。笔记是索引，不是抄本。
